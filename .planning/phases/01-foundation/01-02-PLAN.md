    - "Share" button using `ShareLink(item: "Join my care circle with code: \(code)")`
    - "Generate New Code" button creates a new InviteCode in SwiftData
    - All touch targets minimum 44pt per user decision (SENR-01)
    - Uses system colors per user decision (SENR-02)

    **JoinCircleView.swift** — Caregiver-facing join screen:
    - TextField for entering invite code (auto-uppercase, monospaced)
    - Role picker (MemberRole.allCases)
    - Display name text field
    - "Request to Join" button — looks up code in SwiftData, if valid and unused creates a pending CareTeamMember (with a `isPending: Bool` flag or separate pending model), marks code as used
    - Error state for invalid/used codes

    **PendingRequestView.swift** — Senior-facing approval screen:
    - Shows pending member name and selected role
    - "Approve" button — sets member as active, grants all 4 default permission categories
    - "Reject" button — deletes the pending member
    - Accessible with 44pt touch targets

    **CareTeamListView.swift** — Shows all active care team members:
    - `@Query` to fetch members from the circle

    - Swipe-to-delete with confirmation dialog per user decision ("Removing a member: confirmation dialog, then immediate removal")
    - On removal: delete member from SwiftData, trigger key rotation for ALL categories (since removed member had access)
    - `ContentUnavailableView` for empty state when no team members
    - "Invite Caregiver" button at top/bottom

    **CareTeamTests.swift** — Unit tests:
    - Adding a member to CareCircle persists correctly
    - Removing a member deletes from SwiftData (cascade)
    - Members list fetched with correct role display
