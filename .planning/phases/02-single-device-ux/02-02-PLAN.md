    2. Create AgingInPlace/Features/Medications/ directory.

    3. Create MedicationListView.swift — the main medications screen:
       - Two sections: "Schedules" and "History"
       - Schedules section: @Query MedicationSchedule sorted by drugName, filtered to isActive==true. Each row shows drugName, dose, scheduled time (formatted as "8:00 AM"). Swipe to deactivate (set isActive=false, cancel notification). Toolbar "+" button navigates to MedicationScheduleView.
       - History section: @Query MedicationLog sorted by administeredAt descending. Each row decrypts encryptedPayload to show drugName, dose, time (relative), and author name. Use EncryptionService.open with .medications category. Toolbar "Log Dose" button navigates to LogMedicationView.
       - All touch targets 44pt minimum. Use A11y.minTouchTarget.
       - ContentUnavailableView for empty states in both sections.

    4. Create LogMedicationView.swift — form to log a medication dose:
       - Fields: drugName (TextField), dose (TextField), administeredAt (DatePicker, defaults to now), notes (optional TextEditor)
       - If opened from a schedule (passed as optional MedicationSchedule), pre-fill drugName and dose, set scheduleID
       - On save: JSON-encode {drugName, dose, notes} as Data, call EncryptionService.seal(jsonData, for: .medications), create MedicationLog with encryptedPayload, call context.save()
       - After save: if scheduleID is set, call NotificationService.cancelMissedDoseAlert(for: schedule, onDate: administeredAt) to cancel the missed-dose alert for today
       - Dismiss on save. Validate drugName and dose are non-empty before enabling Save button.

    5. Create MedicationScheduleView.swift — form to create a medication schedule:
       - Fields: drugName (TextField), dose (TextField), scheduledTime (DatePicker with .hourAndMinute), missedWindowMinutes (Picker: 15, 30, 45, 60)
       - On save: Create MedicationSchedule with hour/minute extracted from selectedTime DateComponents. Call context.save().
       - After save: Check notification authorization. If not determined, show NotificationPermissionView first. Then call NotificationService.scheduleMedicationReminder(for: newSchedule) and NotificationService.scheduleMissedDoseAlert(for: newSchedule, onDate: Date()).
       - Dismiss on save.

    6. Create NotificationPermissionView.swift — pre-prompt shown before system notification dialog:
       - Bell icon, "Medication Reminders" title, explanation text about why notifications help
       - "Enable Notifications" button calls NotificationService.requestAuthorization()
       - "Not Now" button dismisses (user can enable later in Settings)
       - Shown only once (track via @AppStorage("notificationPermissionRequested"))
       - Follow the pattern from 02-RESEARCH.md "Requesting Notification Authorization with Pre-Prompt"
